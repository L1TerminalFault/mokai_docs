---
title: "Mokai Command: run"
description: Build and execute runnable binary images safely across cross-platform configurations.
---

The `mokai run` command handles immediate verification, pipeline tracking, and target execution in a single fast sweep. It implicitly constructs a zero-overhead compilation graph through the configuration cache registry, validates artifact footprints, and launches production images without leaving orphan background tasks.

### Usage

```bash
mokai run [target_name] [forwarded_args...]
```

### Operational Architecture

When executed, `mokai run` initiates a rapid resolution lifecycle:
1. **Workspace Sync**: It checks for a local `mokai.toml` file in the execution context path.
2. **Context Resolution**: If no explicit `target_name` parameter is passed, the engine looks into the compiled target configuration arrays for an entry containing `is_default = true` mapped alongside an `Executable` specification rule.
3. **Implicit Graph Builds**: The tool triggers an optimized internal build target check sequence. If any dependency manifest (such as local link mappings like `../fmt`) or source units have shifted, it executes incremental fast-rebuild threads instantly.
4. **Artifact Ingestion**: Once binaries map cleanly inside `./build/debug/` or `./build/release/`, the runner sets up localized process environments and triggers an asynchronous system execution boundary trace.

### Examples

**Running the Default Executable Target:**
If your root configuration has specified a clear default binary unit, trigger execution directly via:
```bash
mokai run
```

**Running an Explicit Target with Direct Arguments:**
To execute a precise secondary binary target and pipe configuration parameters straight down into its runtime parsing array:
```bash
mokai run testfmt --verbose --threads=4
```

### Exit Tracking Trace

If a runtime target encounters an unhandled exception or terminates with an error code, `mokai` traps the underlying shell pipeline failure status directly:

```text
➔ Process exited non-zero runtime status code: 139
```
