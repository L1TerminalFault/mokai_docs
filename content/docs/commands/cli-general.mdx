The general entry-point architecture routes unhandled input flows, outputs baseline telemetry information, and enables command syntax introspection via `mokai help`.

### Global Utility Routines

Running `mokai` with no options prints the standard telemetry greeting layout containing version metrics and localized subcommand strings:

```text
 ◆ mokai v0.0.1a
  Run mokai --help [command] to learn more.

Available commands:
    • build
    • run
    • create
    • add
    • help
```

### Contextual Help Dispatches

The `help` command extracts functional explanations and syntax formatting directly out of static command metadata structures.

```bash
mokai help [subcommand]
```

**Example Output for `mokai help run`:**
```text
 Command: run
 Usage:    mokai run [target_name] [args...]
 Details:  Builds and executes a project binary target, checking default_target
           fallbacks.
```

### Framework Status Codes

Mokai complies with the POSIX standard regarding shell exit behavior, reporting distinct numerical conditions back to parent shells on completion or premature termination:

| Value | Identifier | Condition Context |
| :--- | :--- | :--- |
| `0` | `Success` | Full operational chain execution concluded without runtime issues. |
| `1` | `GeneralFailure` | Compilation dropped with syntax flaws, missing filesystem targets, or cyclic graphs. |
| `2` | `UsageError` | Mismatched arguments, unknown flag structures, or corrupted numeric tokens. |
| `127` | `CommandNotFound` | Invalid subcommand string routed outside of the supported internal maps. |
| `130` | `Interrupt` | Aborted manually via POSIX terminal sig-interrupt boundary (`Ctrl+C`). |
