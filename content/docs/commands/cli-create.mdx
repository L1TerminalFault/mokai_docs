The `mokai create` command initializes a pristine C++ project blueprint instantly. Rather than making external network requests or copying system assets, the engine extracts embedded blueprint layouts straight out of the binary image to write structure, baseline files, and a pre-configured `mokai.toml`.

### Usage

```bash
mokai create [project_name]

```

### Prompt Matrix Workflow

If invoked interactively or with missing context, `mokai create` triggers a raw-terminal selection prompt sequence:

1. **Project Identity**: Mapped directly from your alphanumeric argument string.
2. **Language Specification Selection**: Prompts choice from modern standards up to experimental versions:
`c++11` | `c++14` | `c++17` | `c++20` | `c++23` | `c++26`
3. **Skeleton Blueprints**: Dispatches structured baseline profiles (e.g., `Console Application`, `Static Library Framework`).
4. **Version Control Integration**: Asks whether to initialize a local git tree. If chosen, it executes `git init --initial-branch=main`.

### Layout Output

Running `mokai create application_test` builds out the following workspace shape:

```text
application_test/
├── .git/                # Optional Version Control Tree
├── mokai.toml          # Preconfigured Manifest Context
└── src/
    └── main.cpp        # Baseline Entry Source

```
